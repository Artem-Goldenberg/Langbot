# langbot

`langbot` — небольшой CLI-ассистент основанный на библиотеке LangChain.

## Переменные окружения

Проект автоматически загружает `.env` при старте.

В перменной окружения `OPENAI_API_KEY` должен быть ключ API к openai провайдеру. 
Для opneai-compatible провайдеров указывайте еще `OPENAI_BASE_URL`.

Имя модели задаётся через CLI-флаг `--model`.

## Установка и запуск

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -e .
```

Пример запуска без fallback-модели (при активированном окружении):

```bash
langbot --model gpt-5-mini --character pirate --memory summary
```

## Что умеет бот

- Разные характеры ответа: `friendly`, `professional`, `sarcastic`, `pirate`.
- Классификация входящего сообщения перед ответом (`question`, `task`, `small_talk`, `complaint`, `unknown`).
- Две стратегии памяти: `buffer` и `summary`.
  `buffer` обрезает историю под лимит контекста, `summary` сжимает прошлый диалог в короткую сводку.
- Запоминание сущностей.
  Извлекает устойчивые факты о пользователе в JSON и сохраняет их на всю сессию.
- Стриминг ответа.
- Опциональная fallback-модель.
- Логирование событий, запросов и ответов в файл (внутри `.logs/`).
- Кеш запросов к модели через SQLite.

## Команды в CLI

- `/help` — показать справку
- `/status` — показать текущий характер, тип памяти и число сообщений в истории
- `/character <name>` — переключить характер
- `/memory <strategy>` — переключить стратегию памяти
- `/clear` — очистить историю диалога, сохранив извлечённые сущности
- `/quit` — выйти

Поддерживаемые характеры:

- `friendly`
- `professional`
- `sarcastic`
- `pirate`

Поддерживаемые стратегии памяти:

- `buffer` — хранит историю и выкидывает первые сообщения по токенам, когда контекст переполняется
- `summary` — при переполнении заменяет историю краткой сводкой предыдущего диалога


## Флаги запуска

```bash
.venv/bin/langbot --help
```

- `--character` — выбрать характер
- `--memory` — выбрать `buffer` или `summary`
- `--model` — имя модели
- `--fallback-model` — имя резервной модели, по умолчанию отключена
- `--cache-db` — путь к SQLite-кешу
- `--log-file` — путь к trace-файлу

Что используется по умолчанию:

- характер: `friendly`
- память: `buffer`
- модель: `gpt-5-mini`
- fallback-модель: отключена
- файл кеша: `.cache/langbot.db`
- файл лога: `.logs/DD-MM_HH-MM`
